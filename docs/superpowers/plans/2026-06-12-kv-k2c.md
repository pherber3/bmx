# KV Track K2c Implementation Plan — frozen-subspace generalization (streaming viability)

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Test E5, the load-bearing assumption of the streaming key codec: a
channel subspace fitted on PREFILL keys generalizes to keys written LATER
during generation. If true, the K2b recipe streams via freeze-and-project
(fit V once at prefill; each new key stores r fp16 coefficients + quantized
residual — same asymptotic bpe as the offline codec). If false, measure the
drift cadence so refresh frequency is a number, not a guess.

**Hypothesis (falsifiable):** the subspace is a property of the model/layer
more than of the specific text — frozen-prefill V captures ≥ 90% of the
energy an oracle subspace (fitted on the future tokens themselves) captures,
and the end-to-end frozen codec's attention-score error is within ~20% of the
offline codec's on the same tokens. Kill criterion: large frozen-vs-oracle
gaps or strong decay with token distance ⇒ the streaming design needs
frequent refits and its bpe story weakens.

**Architecture:** one experiment script, no new library code (the projection
is two lines; codecs/metrics/caches all exist). Runs on the committed cache
files — CPU, minutes.

### Task 1: `experiments/k2c_subspace_drift.py` + runs on both caches

**Files:** Create `experiments/k2c_subspace_drift.py`.

tyro Config: cache_path, model_label="", ranks=(8, 16, 32), bits=3, group=64,
fit_frac=0.5, n_blocks=4 (drift curve granularity), kinds=("k_pre", "k").

Per layer × kind × rank, with M = the (S, C) cache matrix (K1 convention):
1. Split tokens: FIT = M[: int(fit_frac·S)], TEST = M[int(fit_frac·S):].
2. Subspaces (right/channel side): V_fit from truncated SVD of FIT
   (`bmx.decomp.lrs.truncated_svd(FIT, r)` returns (Us, V) — V is (C, r));
   V_oracle from truncated SVD of TEST.
3. **Generalization metrics on TEST:**
   - eps_frozen = ‖TEST @ V_fit @ V_fitᵀ‖²_F / ‖TEST‖²_F, eps_oracle same with
     V_oracle; record both + ratio eps_frozen/eps_oracle (the headline).
   - End-to-end codec error: L = TEST @ V V^T, residual quantized with
     `quantize_cache("rtn_channel", TEST − L, bits, group)`; reconstruct;
     compute rel_fro AND `logit_distortion(K_test_true, K_test_hat, Q)` with
     the layer's stored q (reshape (S_test, C) -> (h_kv, S_test, d) per the
     K1 inverse convention) — three arms: frozen V_fit, oracle V_oracle, and
     the OFFLINE codec on TEST (`quantize_cache("lowrank_rtn_channel", TEST,
     bits, group, rank)`) as the reference ceiling.
4. **Drift curve:** fit V on the first S/(n_blocks+... simpler: fit on block 0
   of (n_blocks+1) equal token blocks, record eps captured on each subsequent
   block b=1..n_blocks (one row per block with its token-distance midpoint).
   This answers "does capture decay with distance" and sets refresh cadence.
Row schema: model, layer, kind, rank, segment ("test" rows vs "drift" rows
with block index), eps_frozen, eps_oracle, eps_ratio, rel_frozen, rel_oracle,
rel_offline, logit_frozen, logit_oracle, logit_offline (NaN on drift rows),
plus S_fit/S_test. Parquet via create_run("k2c_subspace_drift", cfg); per-row
prints; end summary per kind: mean/min eps_ratio over layers at each rank,
mean logit_frozen vs logit_offline.

RUN on both committed caches:
`--cache-path results/cache/gpt2_1024.safetensors --model-label gpt2` and
`--cache-path results/cache/llama-3.1-8b_2048.safetensors --model-label
llama-3.1-8b`. Full gate; `git add experiments/k2c_subspace_drift.py
results/k2c_subspace_drift/` (NO commit).

Report the science: eps_ratio range (the ≥0.9 hypothesis), frozen-vs-offline
logit gap (the ≤20% hypothesis), drift-curve shape (flat ⇒ no refresh needed;
decaying ⇒ read the cadence off the curve), and whether k_pre behaves better
than k (it should — RoPE-smearing makes post-RoPE subspaces position-dependent,
which is itself a check of the mechanism: if k_pre is flat where k decays,
that's the RoPE story confirmed a third way).

## Self-review notes
- Drift "fit on block 0" intentionally uses a SHORTER fit window than the
  test-half split — both views matter (half-split = realistic prefill size;
  block-0 fit = pessimistic short-prefill case). The summary should say which
  is which.
- Q probes overlap the TEST segment positions for the Llama cache (q = last
  256 positions of S=2048; TEST = positions 1024+) — that's the realistic
  probe (late queries attending over late keys); note it, don't fix it.
- No new deps, no new library code; `quantize_cache`'s existing svd_factors
  param is NOT used (its Us belongs to the fitted matrix — frozen projection
  is computed inline instead).
