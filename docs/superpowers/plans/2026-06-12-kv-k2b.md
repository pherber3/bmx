`experiments/k2b_cache_ppl.py`. Reuses `bmx.cache.codecs.quantize_cache`,

### Task 1: `src/bmx/cache/ppl_eval.py` + offline tests

**Files:** Create `src/bmx/cache/ppl_eval.py`, `tests/test_ppl_eval.py`.

Public API:

```python
@dataclasses.dataclass
class CacheCodecSpec:
    arm: str            # bmx.cache.codecs.CACHE_ARMS member, or "fp16" (no-op)
    bits: int = 3
    rank: int = 0
    group: int = 64
    seed: int = 0
    pre_rope: bool = False  # quantize k_pre then apply_rope (keys only)

def quantized_prefill_ppl(
    model, input_ids: torch.Tensor, n_prefill: int,
    k_spec: CacheCodecSpec, v_spec: CacheCodecSpec,
) -> dict:
    """Returns {"ppl": float, "bpe_k": float, "bpe_v": float, "n_eval": int}.

    1. Prefill input_ids[:, :n_prefill] with use_cache=True (capturing k_pre via
       the collect-style hooks iff k_spec.pre_rope).
    2. For each layer: build the (S, h_kv*d) matrix (K1 convention) from the
       cache K (or captured k_pre), quantize via quantize_cache, reshape back;
       for pre_rope apply rope_cos_sin/apply_rope before writeback. Same for V
       with v_spec (pre_rope ignored for V). "fp16" spec = identity, bpe 16.0.
    3. Write back into the live cache layers (keys/values, original dtype,
       re-add batch dim), then forward input_ids[:, n_prefill:] with
       past_key_values=cache and labels; ppl = exp(out.loss).
    """
```

Implementation notes:
- Cache surgery on transformers 5.x DynamicCache: `cache.layers[i].keys = new (1, h_kv, S, d)` — VERIFY mutability against the installed version first (if attributes are read-only, fall back to building a fresh DynamicCache via its update() API with the quantized tensors). Get layer count/shapes from the cache itself, not the config.
- The continuation forward must see correct position ids (transformers infers from cache length — verify the loss is over the M continuation tokens only; with labels=continuation ids the internal shift loses 1 token, fine, record n_eval).
- pre_rope path: register the k_pre hooks (reuse/refactor `bmx.cache.collect._register_hooks` — expose what's needed rather than copy-pasting; a small refactor of collect.py to share the hook builders is acceptable and should be test-covered by the existing collect tests staying green).
- fp32/bf16: quantize in fp32, cast back to the cache's native dtype on writeback.

Tests (tiny GPT-2 + tiny Llama from the existing factories' configs, offline):
1. **Identity invariant (load-bearing):** k_spec=v_spec=fp16 gives ppl equal to a plain full-sequence forward's continuation ppl within 1e-3 rel (cache surgery is a no-op; this pins the surgery + position handling).
2. **High-bits ≈ baseline:** bits=8 rtn_token K+V within 2% rel of fp16 ppl.
3. **Garbage hurts:** bits=2 rtn_token ppl > fp16 ppl (strictly).
4. **pre_rope identity:** on tiny Llama, fp16-equivalent pre_rope path (arm fp16 + pre_rope=True must roundtrip k_pre→rope→keys: assert resulting ppl within 1e-2 rel of fp16 baseline — pins the capture/rope/writeback chain).
5. bpe reporting matches quantize_cache's.

Steps: failing tests → implement → pass → full gate → `git add src/bmx/cache/ppl_eval.py tests/test_ppl_eval.py src/bmx/cache/collect.py` (NO commit).

### Task 2: `experiments/k2b_cache_ppl.py` + GPT-2 run

tyro Config: model_name="gpt2", n_prefill=768, n_eval=256, bits=(2,3), rank=32,

Arms table (each row = (k_spec, v_spec), evaluated at each bits b):
1. `fp16` — baseline (run once, bits-independent).

model, arm_k, arm_v, bits, rank, pre_rope, n_prefill, bpe_k, bpe_v, ppl,
dppl_pct = 100·(ppl/ppl_fp16 − 1). Parquet via create_run("k2b_cache_ppl", cfg);
per-row prints; end summary table sorted by dppl_pct per bits.

## Self-review notes
- Streaming codecs, asymmetric quant, 32k contexts: out of scope, recorded as caveats.
- The identity invariant (test 1) is the whole eval's validity anchor — if cache
  surgery or position handling is wrong, every downstream number lies; it must be
  exact, not approximate.
- Judgment calls surfaced: quantized-prefill setting; teacher-forced continuation
  NLL (no sampling); V never pre_rope (RoPE applies to keys/queries only).
